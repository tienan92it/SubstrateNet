Interactive dashboard

The local view, built for humans and structured as the DIKW pyramid: a self-contained, offline dashboard from your SQLite databases. The global view opens on Profile — the synthesized Wisdom layer — and a Map you can drill into. Per-project graphs render domains → concepts/entities → rules. The file dependency graph stays in graph.json for agents.

Build the viewer (once)

The dashboard SPA (Vite + React + react-force-graph) ships as source under dashboard/. Build the bundle once; it compiles to a single self-contained HTML the CLI reuses for every project. 

npm run build:dashboard     # installs + builds dashboard/ -> dashboard/dist
# or build everything (dashboard + CLI):
npm run build:all

Per-project knowledge graph

subnet update rebuilds each project's own dashboard. To build one on demand, the per-project command is still available: 

cd /path/to/your/project
subnet dashboard            # writes .substrate-net/dashboard/index.html + graph.json
subnet dashboard --open     # and open it in your browser

The graph snapshot is inlined into index.html, so it opens directly from disk (file://) with no server. Run subnet update to refresh.

Global dashboard: Profile & Map

subnet global dashboard builds the cross-project view from ~/.substrate-net/global.db and synthesizes the Wisdom layer first. subnet update --global (or subnet global link) populates the zones, skills, and highlights it draws from. 

subnet update --global          # link all projects + rebuild global dashboard
subnet global wisdom            # (re)synthesize competencies, insights, gaps
subnet global dashboard --open  # writes ~/.substrate-net/dashboard/index.html

Profile — the Wisdom layer

The default landing reads top-down through the DIKW pyramid:

  • Wisdom hero — a synthesized headline and narrative: who you are professionally and where your leverage is, over a compact D→I→K→W indicator.
  • Competency map — your skills grouped into 6–8 areas, each graded by proficiency level (novice → expert) and showing its member skills. Replaces a flat skill list with a leveled, deduped view.
  • Insights & principles — evaluated cross-project judgments: patterns in how you work and what you consistently do well.
  • Gaps to close — named knowledge gaps with concrete recommendations — what to learn or document next.

The Wisdom layer is synthesized by a reasoning agent (with a deterministic offline fallback) and is grounded model — kept visibly separate from the structural/stated/corroborated truth of your projects. Industries and portfolio highlights are aggregated mechanically from L2.5/L5 evidence; nothing is invented.

Map — the cross-project wiki

A navigable hierarchy of industry → business domain → tech domain → project. Same-named domains merge across projects, so shared business and technical zones surface as single nodes weighted by how many projects reach them. Click a project node to drill into its knowledge graph; use the breadcrumb to return.

The snapshot

subnet dashboard also writes a standalone graph.json next to the HTML. It is a bounded export from code.db + knowledge.db:

  • knowledge — the human graph: business/tech domain, concept, entity, and fact nodes + their relationships (the dashboard's main view)
  • nodes / edges — the agent-facing file dependency graph (files by layer; symbol-level imports/calls), kept for tooling/MCP
  • domains — industries, portfolio highlights, entities
  • concepts — L3 concepts
  • search — a prebuilt fuzzy index

Sharing

Both artifacts are self-contained. Commit graph.json for teammates, or hand someone the single index.html — it renders offline with no install. (Inspired by Understand-Anything's commit-the-graph workflow.)
